Docker Volume Permissions with SELinux

Thursday, Dec 26, 2019

Unfamiliar with running Docker on a SELinux enabled system, I found myself running into a bunch of file permission errors while creating volumes. 1 mkdir: can't create directory '/data': Permission denied 1 2 3 4 5 $ docker info --format '{{json .SecurityOptions}}' [ "name=seccomp,profile=/etc/docker/seccomp.json", "name=selinux" ] It turns out that this can be resolved by appending the :z flag to the volume mappings in the docker-compose.

docker selinux
